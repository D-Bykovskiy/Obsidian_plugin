# Инструкция для ИИ-Агента: Структура и логика плагина "Monitoring Plugin"

Этот файл содержит карту проекта. Он предназначен для ИИ-ассистентов (например, Antygravity) и разработчиков проекта. Здесь описано, где находится бизнес-логика, архитектурная логика и UI-компоненты плагина, а также в каких случаях следует обращаться к тем или иным файлам.

## 🗂 Структура проекта

```text
Plagin_omis/
├── src/
│   ├── main.ts                   # Главная точка входа, инициализация плагина
│   ├── llm/                      # Модуль интеграции с корпоративной LLM (AI-аналитика)
│   │   └── LLMService.ts
│   ├── outlook/                  # Модуль взаимодействия с почтой Microsoft Outlook
│   │   ├── OutlookService.ts     # TypeScript-обертка для вызова Python-скрипта
│   │   └── fetch_mail.py         # Скрипт на Python (win32com) для MAPI подключения
│   ├── notes/                    # Модуль шаблонизации и работы с файловой системой Obsidian
│   │   └── TemplateManager.ts
│   └── settings/                 # Модуль настроек плагина (UI настроек и хранение конфигурации)
│       └── SettingsTab.ts
├── manifest.json                 # Манифест плагина для Obsidian
├── package.json                  # Зависимости и скрипты сборки
├── tsconfig.json                 # Настройки компилятора TypeScript
└── esbuild.config.mjs            # Скрипт сборки плагина
```

## 🧭 Когда и куда смотреть агенту?

### 1. Архитектурная логика (Ядро и Связи)
**Файлы:** `src/main.ts`, `manifest.json`
* **Смотри сюда, если:**
  * Нужно добавить новую команду в Command Palette (через `this.addCommand`).
  * Нужно добавить новую кнопку (Ribbon Icon) на боковую панель Obsidian.
  * Нужно добавить новые события при загрузке плагина (`onload`) или его выключении (`onunload`).
  * Требуется изменить зависимости между сервисами (если один сервис должен передавать данные в другой).
  * Обновляется версия плагина или требуемая версия Obsidian (`minAppVersion`).

### 2. Бизнес-логика: Маршрутизация и Шаблоны инцидентов (Создание заметок)
**Файл:** `src/notes/TemplateManager.ts`
* **Смотри сюда, если:**
  * Нужно поменять внешний вид или структуру создаваемой Markdown-заметки (Frontmatter/YAML-шапку, заголовки, разметку).
  * Нужно изменить алгоритм нейминга файлов (как называется файл: по дате, по теме письма и т.д.).
  * Появилось требование сохранять заметки в строго определенную папку внутри хранилища (Vault) Обсидиана.

### 3. Бизнес-логика: Работа с почтой (Microsoft Outlook)
**Файлы:** `src/outlook/fetch_mail.py`, `src/outlook/OutlookService.ts`
* **Смотри сюда, если:**
  * Изменились правила извлечения почты (например, нужно парсить не `Inbox`, а 특정ную подпапку).
  * Сломалась интеграция с Windows (`win32com` перестала возвращать данные). Вносить изменения сначала в Python-скрипт.
  * Требуется фильтровать почту (выбирать письма только от определенных отправителей или с определенными темами).
  * Нужно изменить структуру JSON, который Python-скрипт возвращает обратно в плагин.

### 4. Бизнес-логика: AI и Суммаризация инцидентов (LLM)
**Файл:** `src/llm/LLMService.ts`
* **Смотри сюда, если:**
  * Нужно поменять **system prompt** (инструкции для корпоративной нейросети, как обрабатывать текст инцидента).
  * Необходимо добавить новые фичи ИИ (например, генерация еженедельного отчета, перевод текста, извлечение задач в формате TODO).
  * Изменились конечные точки (URL API) для подключения к LLM (форматы OpenAI).
  * Вводятся новые параметры для LLM (например, изменение `temperature`, ограничений `max_tokens` или добавление потокового ответа).

### 5. Пользовательский интерфейс: Настройки и UI
**Файл:** `src/settings/SettingsTab.ts`
* **Смотри сюда, если:**
  * Пользователю требуется дать возможность настраивать новые переменные (добавить поле ввода, ползунок, переключатель).
  * Обновляются ключи API, пути или интервалы опроса почты.
  * Нужно изменить тексты подсказок (descriptions) на странице настроек плагина.

## 🛠 Паттерн взаимодействия слоев

При выполнении любых доработок учитывайте текущий архитектурный паттерн (Data Flow):
1. Событие (клик по кнопке или хук таймера в `main.ts`).
2. Запрос в Outlook -> `OutlookService.ts` вызывает `fetch_mail.py` через `exec()`. Данные возвращаются в виде массива JSON.
3. Отправка сырых данных письма в LLM -> `LLMService.ts` прогоняет текст через системный промпт нейросети.
4. Создание файла -> `TemplateManager.ts` объединяет результаты писем и выжимки от LLM, записывая их в Markdown внутри локального хранилища Obsidian.
