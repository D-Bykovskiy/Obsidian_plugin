# Инструкция по установке и настройке Monitoring Plugin

**Monitoring Plugin** — это инструмент для автоматизации работы с инцидентами прямо в Obsidian. Он забирает письма из Outlook, делает их краткий пересказ с помощью корпоративного ИИ и создает удобную аналитическую панель управления.

---

## 🏗 Шаг 1: Подготовка компьютера

Плагин работает в связке с Microsoft Outlook и Python. Перед установкой проверьте:

1.  **Windows и Outlook**: Должен быть установлен и запущен **классический Outlook** (не веб-версия) с вашей рабочей почтой.
2.  **Python**: Убедитесь, что установлен Python 3.10 или выше.
    *   *Как проверить:* В терминале введите `python --version`.
3.  **Установка необходимых библиотек**:
    Откройте терминал (PowerShell или CMD) и выполните:
    ```bash
    pip install pywin32
    ```
    > [!IMPORTANT]
    > **Если команда `pip install` прошла успешно, но плагин всё равно выдает ошибку**, выполните команду для принудительной регистрации библиотек в системе:
    > ```bash
    > python -m pywin32_postinstall -install
    > ```

---

## 📂 Шаг 2: Установка плагина в Obsidian

1.  Откройте папку вашего хранилища (Vault) в проводнике.
2.  Перейдите в скрытую папку `.obsidian/plugins/`.
3.  Создайте там папку `monitoring-plugin` и скопируйте в неё следующие файлы:
    *   `main.js`
    *   `manifest.json`
    *   `styles.css`
    *   Папку `outlook/` (внутри должен быть `fetch_mail.py`).
4.  В Obsidian перейдите в **Settings** -> **Community Plugins**.
5.  Нажмите кнопку обновления и включите **Monitoring Plugin**.

---

## ⚙️ Шаг 3: Настройка ИИ и Почты

Перейдите в **Settings** -> **Monitoring Plugin** и заполните поля:

1.  **LLM API Key**: Вставьте ваш API-ключ.
2.  **LLM Base URL**: URL корпоративного шлюза ИИ (обычно уже стоит по умолчанию).
3.  **Incidents Folder**: Название папки для заметок (например, `incidents`). Она создастся автоматически.
4.  **Outlook Folder**: Имя папки в Outlook для сканирования (обычно `Inbox`).

## 🚀 Как пользоваться

После установки на левой панели появятся 3 иконки:

### 1. 📊 Главная панель управления (Dashboard)
Показывает общую статистику и трекера. В верхней части панели доступны три вкладки:
*   **Дашборд**: Карточки статусов, отслеживаемые темы и таблицы свежих инцидентов.
*   **Канбан**: Визуальная доска задач и проектов. Перетаскивайте карточки между колонками "Ожидание", "В работе" и "Завершено" для смены статуса.
*   **Календарь**: Линейный график времени (Timeline), показывающий сроки выполнения задач и длительность проектов на текущей неделе.
*   **Кнопка "Сформировать отчет"**: ИИ проанализирует инциденты и создаст для вас аналитическую записку.

### 2. 💬 Корпоративный ИИ Чат
Ваш персональный помощник. Можно задавать вопросы по инцидентам или просить написать ответ на письмо прямо в боковой панели.

### 3. ✉️ Импорт почты
Нажмите на иконку конверта, чтобы запустить сканирование.
*   **Фильтрация**: Чтобы плагин не импортировал всё подряд, откройте файл `Dashboard.md` и в блоке `tracked_subjects` укажите ключевые слова (например, `Проект Альфа`, `Ошибка БД`).

### 4. 🛠 Панель управления в заметке
В каждой созданной заметке (инциденте или задаче) сверху доступен блок управления:
*   **Срок и Приоритет**: Установка дедлайнов и важности (1-5).
*   **Статусы**: Быстрое переключение между *План / В работе / Готово*.
*   **Инструменты**:
    *   `➕ Задачу` — создание подзадачи, связанной тегом с текущей заметкой.
    *   `🔗 Ресурс` — прикрепление внешних ссылок или локальных файлов.
    *   `🏷️ Тег` — управление метками через умный поиск.
*   **Архивация**: Кнопки для быстрого переноса заметки в папку "Архив", "Корзина" или полное удаление.

## ❓ Решение проблем

*   **Ошибка "Failed to execute Outlook python script"**:
    *   Убедитесь, что Outlook **запущен**.
    *   Проверьте, установлена ли библиотека: `pip install pywin32`.
    *   Если не помогло, запустите терминал **от имени администратора** и выполните: `python -m pywin32_postinstall -install`.
*   **Иконки не появились?** Перезагрузите Obsidian (Ctrl+R).
*   **ИИ не отвечает?** Проверьте API-ключ и наличие интернета/VPN (если требуется для корпоративной сети).
