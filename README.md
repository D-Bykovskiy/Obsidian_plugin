# Monitoring Plugin для Obsidian

**Monitoring Plugin** — это мощный инструмент для руководителей отделов мониторинга, который превращает Obsidian в единую панель управления инцидентами. Он автоматически получает входящие письма об инцидентах из Microsoft Outlook, обрабатывает их с помощью корпоративной нейросети (LLM), отслеживает цепочки писем (треды) и автоматически создает или обновляет Markdown-заметки со сводками (Rolling Summaries).

---

## 📋 Особенности и возможности

1. **Интеграция с корпоративной LLM (по умолчанию Beeline API)** — анализ писем на лету, саммаризация инцидентов и извлечение ключевых задач на русском языке.
2. **Отслеживание тредов (Rolling Summary)** — плагин группирует цепочки писем (игнорируя приставки `Re:`, `Fwd:`) и обновляет уже существующую заметку инцидента, переписывая саммари с учетом новых данных и сохраняя лог переписки.
3. **Фильтрация по темам (Dashboard Note)** — вы можете указать список ключевых слов и тем инцидентов в отдельной заметке. Плагин будет игнорировать спам и сканировать только то, что важно.
4. **Умное кэширование** — плагин запоминает уникальные идентификаторы (`EntryID`) обработанных писем, чтобы не анализировать одно и то же письмо дважды.
5. **Панель управления задачами** — внутри каждой заметки инцидента встроена интерактивная панель для управления сроками, статусами, приоритетами и ресурсами.
6. **Менеджер ресурсов** — быстрое добавление ссылок на веб-сайты или локальные файлы прямо в заметку в виде стильной таблицы материалов.
7. **Система тегов** — выбор тегов из существующих в базе Obsidian или создание новых через удобное окно поиска.
8. **Декомпозиция задач** — создание подзадач из текущей заметки с автоматическим связыванием их через теги.
9. **Управление жизненным циклом** — кнопки быстрого архивирования, переноса в корзину или удаления заметок.
10. **Режим локального тестирования (Mock Mode)** — возможность запускать и проверять плагин на домашнем ПК без установленного Outlook и доступа к корпоративной сети (LLM).

---

## 🛠 Требования (Prerequisites)

Плагин использует прямое взаимодействие с десктопными приложениями Windows:
1. **Windows** и **Microsoft Outlook** (должен быть запущен и авторизован в вашей корпоративной записи).
2. **Python 3.7+**. Убедитесь, что Python добавлен в системную переменную `PATH`.
3. Библиотека `pywin32` для интеграции с Outlook. Откройте командную строку и выполните:
   ```bash
   pip install pywin32
   ```

## ⚙️ Настройка конфигурации

После установки и включения плагина, перейдите в **Settings -> Monitoring Plugin**:

1. **LLM Base URL**: `https://api.ai.beeline.ru/api/v3` (по умолчанию)
2. **LLM API Key**: Вставьте ваш токен `YOUR-API-KEY`.
3. **LLM Model**: Укажите модель (по умолчанию `llm-medium-moe-instruct`).
4. **Outlook Folder Name**: Имя папки в Outlook, откуда плагин забирает письма (`Inbox`, `Корзина` и т.д.).
5. **Dashboard Note Name**: Название заметки в вашем Obsidian (по умолчанию `Dashboard.md`), в которой будет храниться список отслеживаемых тем.
6. **Clear Scanned Emails Cache**: Кнопка для ручного сброса памяти плагина, чтобы пересканировать письма заново.

### Настройка Dashboard.md (Фильтрация писем)
Создайте в корне вашего хранилища файл `Dashboard.md`. Добавьте в его начало YAML блок (Properties) со списком тем:

```yaml
---
tracked_subjects:
  - Падение сервера БД
  - Ошибка авторизации в биллинге
  - Релиз
---
```
*Если массив `tracked_subjects` не пуст — плагин будет обрабатывать ТОЛЬКО письма, тема которых содержит эти слова. Если этот файл удалить или оставить массив пустым, плагин будет анализировать ВСЕ входящие письма.*

### Режим разработки и тестирования (Development / Testing)
В самом низу настроек есть переключатели:
- **Use Mock Email Data**: Отключает реальный Outlook. При импорте плагин загрузит 3 тестовых письма (имитируя переписку).
- **Use Mock LLM**: Отключает реальные сетевые запросы к API Beeline. Плагин будет моментально выдавать заранее заготовленный тестовый текст. Замечательно подходит для работы из дома.

## 🚀 Как использовать плагин

На боковой панели Obsidian появится **иконка Письма** (📧) "Импортировать почту" (или команда в Command Palette: `Обновить почту (Import Mail)`).

**Алгоритм работы:**
1. Плагин запрашивает последние 50 писем (или берет тестовые, если включен Mock-режим).
2. Отсеивает те, что уже есть в кэше (`scannedEmailIds`). 
3. Если заметок по теме письма еще нет — плагин обращается к LLM и **создает новый инцидент**.
4. Если заметка с такой темой (`conversation_topic`) уже существует — плагин берет старое саммари, добавляет к нему новое письмо и просит LLM **сделать обновленную выжимку (Rolling Summary)**. Затем перезаписывает заметку, сохраняя историю переписки внизу файла.

Заметки об инцидентах автоматически получают тег `#incident` и нужные метаданные (Properties) для удобного поиска.
